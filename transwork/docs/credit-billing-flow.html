Gressio Credit and Billing Flow

Interactive map of the current setup: internal quota remains the billing ledger, USD-equivalent value is derived from QuotaPerUnit, and Gressio credits are the user-facing display/input unit at 100 credits per USD-equivalent.

The Current Relationships

The database still stores integer quota units. The server converts those quota units into USD-equivalent value with QuotaPerUnit. Transwork then converts USD-equivalent value into user-facing Gressio credits.

1 USD-equivalent = 500,000 internal quota units 1 USD-equivalent = 100 Gressio credits 1 Gressio credit = 5,000 internal quota units 1 Gressio credit = $0.01 USD-equivalent 1 internal quota unit = 0.0002 credits = $0.000002 USD-equivalent
Core ratio: common.QuotaPerUnit Transwork ratio: CREDITS_PER_USD = 100 Balance display floors fractional credits

Insufficient Credit Check

1
Pre-consume reads user quota. service/pre_consume_quota.go checks model.GetUserQuota. This is internal quota, not display credits.
2
If user quota is zero or below, reject. The relay returns HTTP 403 with ErrorCodeInsufficientUserQuota. The client may show this as insufficient credits.
3
If balance cannot cover the reserve, reject. The comparison is quota-vs-quota. Credits are display math only.
4
If allowed, reserve is deducted, then settled. Failure refunds quota; success adjusts by actual usage quota. UI displays the resulting quota as credits.

End-to-End Flow

A
User chooses a top-up amount. Presets come from payment_setting.amount_options. The payment controllers treat the amount as USD-equivalent purchase units for standard providers. The credit equivalent is amount × 100.
B
Server calculates payment cash. Epay/Waffo use configured price, top-up group ratio, and optional discount. Stripe has its own unit price path. These settings decide how much real money is charged; they do not change the 100 credits per USD-equivalent display relationship.
C
Payment callback completes the order. The pending TopUp row is marked success and the user quota is increased. Most providers add amount * QuotaPerUnit; provider-specific paths can differ. After this, shared formatters display the new quota as credits.
D
Shared user-facing quota display shows credits. Web quota formatters and the desktop auth-check display convert quota / QuotaPerUnit * 100. Balance-style display floors the result so the UI does not overstate spendable credits.
E
API usage spends quota. Relay billing calculates quota from model price or ratio, token usage, group ratio, and tiered billing rules where configured. Settlement updates user/token quota and logs usage.

Important Answer

The Gressio server currently has the core USD-equivalent ratio as a real server setting/constant: QuotaPerUnit. The 100 credits per USD-equivalent behavior is a Gressio display convention implemented in Transwork overlay helpers and wired into shared display formatters, not a replacement for internal billing quota.

Changing QuotaPerUnit changes how much internal quota represents one USD-equivalent. Changing CREDITS_PER_USD changes only the user-facing credit scale. With the current defaults, $49.71 USD-equivalent is 24,855,000 quota units and displays as 4,971 credits.
